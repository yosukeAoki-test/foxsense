## 3. 子機の電力収支（電池寿命）※ここが再計算の主眼

**電源: 単三アルカリ2本直列(≈2500mAh実用) → TPS63020(3.3V,PFM) → C3+E220。サイクル20分(方式B同期時)。**

### 3-0. 単三アルカリ2本＋TPS63020 の実収支（2026-07-08確定）
負荷側(3.3Vレール):
- スリープ(1200s中ほぼ): C3 deep-sleep 44µA + E220 Mode3 2µA + SHT3x 0.2µA ≈ **46µA**
- 起床1回(~3s): 起動/測定~1.5s(≈25mA) + TX~0.3s(C3+E220=65mA) + ACK受信~1.5s(C3+E220=31mA) ≈ **100mA·s/回**
  - 3回/h → 300mA·s/h ≈ 0.083mA平均
- **負荷側 合計 ≈ 0.13mA @3.3V**

電池側に換算(TPS63020のIq＋変換損):
- スリープ負荷分: 46µA×(3.3/3.0)/0.80 ≈ 63µA
- **TPS63020 Iq(power-save): ≈35µA**  ← 増分の主因
- 起床負荷分(効率~88%): ≈0.11mA平均
- 電池電圧監視の分圧(470k×2=940kΩ, 生電池直結・常時ON): 3.0V/940k ≈ **+3µA**(寿命−1.5%、A0-GND間100nFでADC精度確保)
- **電池側 合計平均 ≈ 0.2mA**（負荷0.13→電池0.2mA、変換で約+40%。分圧3µAは誤差内）

寿命(単三アルカリ2本, 実用2500mAh):
- 2500mAh ÷ 0.2mA ≈ 12,500h ≈ **理想 約1.4年**
- 低温(0℃以下でアルカリは容量激減)・自己放電込みの現実値 **~1〜1.2年**
- ※屋外冬季を重視するならリチウム乾電池(3000mAh・低温◎)で~1.5年に伸ばせる

**要注意**: TPS63020は必ず**PS/SYNC=Low(power-saveモード)**。SYNC固定だとIqが跳ね寿命が数分の1。支配項はスリープ時の **C3 deep-sleep 44µA + TPS Iq 35µA ≈ 80µA** で、ここが寿命の主レバー(要実機実測)。

---

### 参考: 旧 18650 3000mAh 想定の比較（変換器なし直結の理論値）
前提: 送信サイクル10分、電池 18650 = **3000mAh**、SHT3x測定エネルギーは無視できる。

### シナリオA — 現行v1ファーム（方式A: E220常時RX + C3 間欠listen/light-sleep）
現状の `foxsense-lora-child/` はE220を**Normalモード=常時受信**にしている。
- E220 RX 連続: **11mA**
- C3平均: listen 1.5s@20mA + light-sleep 8s@1mA / 9.5s周期 → ≈ **4.0mA**
- TX: 10分に1回・0.2s@45mA → ≈0.003mA（無視）
- **合計 ≈ 15mA(連続)** → 3000mAh ÷ 15mA ≈ 200h ≈ **約8日**（2000mAhなら約5.5日）

→ **電池駆動には不適**。E220の常時RX(11mA)が支配的。

### シナリオC — 中間案（方式A維持 + E220 WOR受信 + C3 deep-sleep）
E220の**WOR受信モード(Mode2, 空中起動)**で間欠sniff、C3はdeep-sleepしE220割込で起床。
- E220 WOR平均（WORサイクル2s, 推定）: ~**0.5-2mA**（※要実測）
- C3 deep-sleep 44µA
- **合計 ≈ 1-2mA** → 3000mAh ÷ 1.5mA ≈ 2000h ≈ **約80日（~3ヶ月）**

### シナリオB — 推奨（方式B: 子機起点プッシュ, E220&C3ともdeep-sleep）
子機は自タイマで10分ごとに起床→測定→送信→即スリープ。E220は送信前後以外**深いスリープ2µA**。
- スリープ中(599/600s): E220 2µA + C3 44µA ≈ **46µA**
- 起床1回(~2s): C3 20mA×2s + TX 45mA×0.2s + (任意ACK受信 11mA×1s) ≈ **60mA·s**
  - 1時間=6回 → 360mA·s = 0.1mAh/h → 平均 **0.1mA**
- **合計平均 ≈ 0.15mA** → 3000mAh ÷ 0.15mA ≈ 20,000h ≈ **約2年**（2000mAhで~1.5年）
  - 実効は自己放電等で短縮するが、桁として **「日 → 年」**。

### まとめ（方式別・電池寿命）
| 方式/電源 | 平均電流 | 電池寿命 | 備考 |
|---|---:|---:|---|
| A: 現行v1 (E220常時RX, 18650) | ~15mA | **~8日** | 動作優先の初期実装 |
| C: 方式A+WOR+deep-sleep (18650) | ~1.5mA | **~3ヶ月** | 方式Aのまま省電力化 |
| B: 子機起点+deep-sleep (18650直結・理論) | ~0.15mA | ~1.5-2年 | 変換器なし理論値 |
| **B: 単三アルカリ2本＋TPS63020（採用）** | **~0.2mA(電池側)** | **~1〜1.4年** | 現場交換容易・変換損+Iq込み |

## 4. 親機の電力収支

親機はLTE(SIM7080G)が支配的で、E220追加の影響は小さい。
- **スリープ中**: E220をM0=M1=HIGH(Mode3)で保持 → **+2µA**のみ（SIM7080G PSM + ESP32-S3 deep-sleepに対し無視できる）。
- **起床中(~30-60s/サイクル)**: E220でwake送信・収集 → RX11mA〜TX110mAが数十秒。LTE送信(平均100-200mA)と同オーダーで、全体を大きく変えない。
- 親機は10分サイクルでLTE通信する前提上、**外部電源 or 大容量電池/ソーラー**運用が現実的。E220化による親機側の電力インパクトは軽微。

## 5. 結論・実装状況

**方式B（子機起点プッシュ + deep-sleep）を実装済み**（2026-07-01）。子機電源は**単三アルカリ2本＋TPS63020(3.3V,PFM)**を採用（2026-07-08確定）。電池側平均 **~0.2mA**、寿命 **~1〜1.4年**（旧方式A常時RXの約8日から大幅改善）。18650直結より変換損＋Iqで短めだが現場交換が容易。屋外冬季重視ならリチウム乾電池で~1.5年。

### 実装した動作モデル
- **親子とも20分毎にdeep-sleep起床**。
- **子機**: 起床→測定→DATA送信→DATA_ACK待ち(3回リトライ)。ACK有→20分sleep / ACK無→90s短sleepで親の受信窓をハント。deep-sleep中はE220もMode3(2µA)。
- **親機**: 毎回60s受信窓で子機DATA収集＋ACK。計測をRTCに蓄積し、**LTE送信は3回に1回(≒1時間)** まとめて送信。非LTE起床はモデム未起動。
- **親機の副次効果**: LTEを1時間に1回に間引くことで、親機側のモデム稼働も1/3になり親機電力も削減（従来は20分毎→現行60分毎）。

### ランデブー（親子の窓合わせ）
時刻同期は使わず、**子機のACK有無ハント**で収束。ACHが取れない間は90s短sleepで位相を掃引し、親の60s窓に入ると以後20分sleepで同位相を維持（RC誤差~5%=20分で~60s→窓幅60sでほぼ吸収）。

### 残注意
- 数値はデータシート公称ベースの概算。**実機で平均電流を実測して確定**（特にC3 deep-sleepの基板込み値、13dBm時TX電流、E220起床/設定時間）。
- 収束が悪い場合は親機の窓(CHILD_RESPONSE_TIMEOUT=60s)や子機RESYNC(90s)を調整。将来はACKに「次窓までの秒数」を載せて明示同期も可能。
